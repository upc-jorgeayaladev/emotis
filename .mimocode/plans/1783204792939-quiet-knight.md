# Plan de Implementación: Emotis Social Network

## Resumen
Red social de regalos y emociones con stack MERN moderno. MVP completo con autenticación, posts, stories, calendario, wishlist, insignias y comunidad Q&A.

## Fase 1: Configuración del Proyecto (Día 1)

### 1.1 Estructura Monorepo
```
emotis/
├── frontend/          # React + Vite + Tailwind
├── backend/           # Express + MongoDB + Cloudinary
├── .gitignore
├── logo.png
└── README.md
```

### 1.2 Inicializar Backend
- `npm init` en `/backend`
- Dependencias: express, mongoose, dotenv, cors, helmet, morgan, bcryptjs, jsonwebtoken, multer, cloudinary, zod
- Dev: nodemon
- Crear estructura:
  ```
  backend/src/
  ├── config/          # db.js, cloudinary.js
  ├── controllers/
  ├── middlewares/     # auth.js, error.js
  ├── models/
  ├── routes/
  ```

## Fase 2: Backend Core (Días 2-3)

### 2.1 Base de Datos - Modelos
**User.js:**
```javascript
{
  name: String,
  username: { type: String, unique: true },
  email: { type: String, unique: true },
  password: String, // bcrypt hash
  avatar: { secure_url: String, public_id: String },
  cover: { secure_url: String, public_id: String },
  bio: String,
  followers: [{ type: ObjectId, ref: 'User' }],
  following: [{ type: ObjectId, ref: 'User' }],
  badges: [String],
  wishlist: [{ name: String, url: String }],
  createdAt: Date
}
```

**Post.js:**
```javascript
{
  user: { type: ObjectId, ref: 'User' },
  description: String,
  images: [{ secure_url: String, public_id: String }],
  reactions: {
    emotion: [ObjectId],    // ❤️ Me emocionó
    inspire: [ObjectId],    // 🎁 Me inspira
    cry: [ObjectId],        // 🥹 Me hizo llorar
    want: [ObjectId],       // 😍 Lo quiero
    perfect: [ObjectId]     // 👏 Excelente idea
  },
  comments: [{ type: ObjectId, ref: 'Comment' }],
  occasion: String,         // cumpleaños, boda, etc.
  giftProduct: String,      // producto relacionado
  createdAt: Date
}
```

**Story.js:**
```javascript
{
  user: { type: ObjectId, ref: 'User' },
  media: { secure_url: String, public_id: String, type: String },
  text: String,
  poll: { question: String, options: [{ text: String, votes: [ObjectId] }] },
  expiresAt: Date,          // 24 horas
  createdAt: Date
}
```

**Comment.js:**
```javascript
{
  user: { type: ObjectId, ref: 'User' },
  post: { type: ObjectId, ref: 'Post' },
  content: String,
  createdAt: Date
}
```

**Notification.js:**
```javascript
{
  to: { type: ObjectId, ref: 'User' },
  from: { type: ObjectId, ref: 'User' },
  type: String,             // follow, reaction, comment, birthday
  post: ObjectId,
  read: Boolean,
  createdAt: Date
}
```

**Calendar.js:**
```javascript
{
  user: { type: ObjectId, ref: 'User' },
  title: String,
  date: Date,
  type: String,             // birthday, anniversary, other
  reminder: Boolean,
  reminderDays: Number,
  createdAt: Date
}
```

### 2.2 Controllers Principales
- `authController.js`: register, login, getMe
- `userController.js`: getProfile, updateProfile, followUser, unfollowUser, searchUsers
- `postController.js`: createPost, getFeed, getPostsByUser, deletePost
- `storyController.js`: createStory, getStories, deleteStory
- `commentController.js`: addComment, getComments
- `calendarController.js`: addEvent, getEvents, deleteEvent
- `notificationController.js`: getNotifications, markAsRead

### 2.3 Middlewares
- `auth.js`: Verificar JWT
- `error.js`: Manejo centralizado de errores
- `upload.js`: Configuración de Multer para Cloudinary

### 2.4 Rutas
```
POST   /api/auth/register
POST   /api/auth/login
GET    /api/auth/me

GET    /api/users/:username
PUT    /api/users/profile
POST   /api/users/follow/:id
DELETE /api/users/follow/:id

GET    /api/posts
POST   /api/posts
DELETE /api/posts/:id

POST   /api/stories
GET    /api/stories
DELETE /api/stories/:id

POST   /api/posts/:id/comments
GET    /api/posts/:id/comments

GET    /api/calendar
POST   /api/calendar
DELETE /api/calendar/:id

GET    /api/notifications
PUT    /api/notifications/read
```

### 2.5 Validaciones (Zod)
- Register: name (3+), username (3+), email, password (8+)
- Login: email, password
- Post: description (opcional), images (max 5)

## Guía de Configuración de Servicios

### MongoDB Atlas (Base de Datos)

1. **Crear cuenta**: Ve a [mongodb.com/atlas](https://www.mongodb.com/atlas) y regístrate gratis

2. **Crear clúster gratuito**:
   - Click "Build a Database"
   - Selecciona "FREE" (M0 Sandbox)
   - Elige región más cercana (US East o similar)
   - Click "Create"

3. **Configurar acceso**:
   - Ve a "Database Access" → "Add New Database User"
   - Crea usuario con contraseña fuerte
   - En "Network Access" → "Add IP Address"
   - Selecciona "Allow Access from Anywhere" (0.0.0.0/0)

4. **Obtener URI de conexión**:
   - Ve a "Database" → "Connect"
   - Selecciona "Connect your application"
   - Copia la URI (parece: `mongodb+srv://user:pass@cluster0.xxxxx.mongodb.net/...`)
   - Reemplaza `<password>` con tu contraseña real

5. **Variable de entorno**:
   ```
   MONGODB_URI=mongodb+srv://usuario:contraseña@cluster0.xxxxx.mongodb.net/emotis?retryWrites=true&w=majority
   ```

---

### Cloudinary (Almacenamiento de Imágenes)

1. **Crear cuenta**: Ve a [cloudinary.com](https://cloudinary.com) y regístrate gratis

2. **Obtener credenciales**:
   - En el Dashboard verás:
     - **Cloud Name**: `dxxxxxxxx`
     - **API Key**: `xxxxxxxxxxxxxxxxxxxx`
     - **API Secret**: `xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx`

3. **Variables de entorno**:
   ```
   CLOUDINARY_NAME=dxxxxxxxx
   CLOUDINARY_KEY=xxxxxxxxxxxxxxxxxxxx
   CLOUDINARY_SECRET=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```

4. **Configuración en backend**:
   ```javascript
   // src/config/cloudinary.js
   const cloudinary = require('cloudinary').v2;

   cloudinary.config({
     cloud_name: process.env.CLOUDINARY_NAME,
     api_key: process.env.CLOUDINARY_KEY,
     api_secret: process.env.CLOUDINARY_SECRET
   });

   module.exports = cloudinary;
   ```

---

### JWT Secret

Genera un secreto seguro. Ejemplo con Node:
```bash
node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"
```


## Notas Importantes

1. **Cloudinary**: Es OBLIGATORIO para imágenes. Nunca guardar en MongoDB.
2. **JWT**: Almacenar en localStorage (o cookie HTTP-only para más seguridad).
3. **bcrypt**: SIEMPRE hashear contraseñas.
4. **Variables de entorno**: NUNCA subir .env a git.
5. **Validar todo**: Tanto frontend (Zod) como backend.
